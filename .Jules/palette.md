## 2026-03-30 - [Convert Interactive Divs to Semantic Buttons]
**Learning:** The floating dock navigation component relied heavily on `<div class="dock-item">` elements with `onclick` handlers for interactivity. This pattern completely broke keyboard accessibility (no focus states or ability to activate via enter/space) and lacked necessary context for screen readers. Using visual focus rings coupled with `aria-label`s for icon-only buttons dramatically improves accessibility while retaining visual aesthetics.
**Action:** When implementing custom interactive toolbars or lists, always use semantic `<button>` elements instead of `div`s, add `role="toolbar"`, use `role="separator"` for dividers, hide purely decorative nested icons with `aria-hidden="true"`, and provide proper accessible names with `aria-label`.
